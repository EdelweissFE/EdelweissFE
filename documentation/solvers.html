Nonlinear solvers

Currently, EdelweissFE provides

  • a nonlinear implicit static solver (NIST),

  • a parallel nonlinear implicit static solver (NISTParallelForMarmotElements),

  • and a parallel arc length solver (NISTPArcLength).

Choose the solver in the *job definition:

*job, name=myJob, domain=2d, solver=NISTParallel
Currently available:

NIST - Nonlinear Implicit Static

A standard nonlinear, implicit static solver.

class fe.solvers.nonlinearimplicitstatic.NIST(jobInfo, journal)

This is the Nonlinear Implicit STatic – solver. Designed to interface with Abaqus UELs Public methods are: __init__(), initializeUP() and solveStep(…). OutputManagers are updated at the end of each increment.

Apply the dirichlet bcs on the Residual vector -> is called by solveStep() before solving the global sys.

computeElements(elements, Un1, dU, P, K, F, increment)

NISTPArcLength - Nonlinear Implicit Static - Arc length

(Parallel) Arc Length Solver, based on the proposed approach in Jirásek/Bažant 2001. Replaces the NewtonRaphson scheme of the NISTParallel Solver.
